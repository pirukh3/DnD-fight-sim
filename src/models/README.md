# Models

Data models representing game entities.

Examples of models to implement here:
- **Character** - Player characters with attributes, classes, and abilities
- **Monster** - Creatures with stat blocks (HP, AC, attacks, etc.)
- **Spell** - Spell definitions with effects, ranges, and components
- **Item** - Weapons, armor, and equipment with properties
- **Condition** - Status effects like poisoned, stunned, frightened, etc.
